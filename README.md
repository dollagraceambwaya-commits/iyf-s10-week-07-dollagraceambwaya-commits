## Project Description

This repo showcases **JavaScript best practices** through modular mini‑projects.  
Each folder represents a challenge or app that demonstrates clean code, debugging, refactoring, and storage APIs.  
The goal is to build confidence in writing maintainable, professional JavaScript code.

---

## Technologies Used

- **HTML5** — structure and markup
- **CSS3** — styling and responsive layouts
- **JavaScript (ES6+)** — core logic, DOM manipulation, async/await
- **LocalStorage & SessionStorage** — persistence of user data
- **ESLint + Prettier** — enforcing clean code formatting
- **Git & GitHub** — version control and repo organization

---

## Features

- **Theme Persistence**: Toggle light/dark mode with saved preferences.
- **Recent Searches**: Store and display last 5 searches.
- **Form Auto‑Save**: Preserve form input values automatically.
- **Refactor Challenge (Day 4)**: Improve readability and maintainability of existing code.
- **Code Review Challenge (Day 5)**: Weather Dashboard analysis with REVIEW.md documentation.
- **Notes App**: Create, delete, and persist notes.
- **Shopping Cart**: Add/remove items, manage state, display product images.
- **Todo List**: CRUD operations with persistent storage.
- **Debug Practice**: Exercises for fixing broken JavaScript code.

---

## Challenges Faced

- Handling **ESLint warnings** (unused variables, global scope conflicts).
- Debugging **storage persistence** across multiple projects.
- Managing **repo structure** to keep projects modular and clean.
- Refactoring inline styles into **CSS classes** for maintainability.
- Implementing **error handling** for API calls and geolocation.

---

## Lessons Learned

- **Consistency matters**: ESLint + Prettier enforce clean, readable code.
- **Separation of concerns**: Splitting logic into functions (fetch, render, storage) makes debugging easier.
- **Refactoring improves maintainability**: Removing unused variables and replacing inline styles reduced errors.
- **LocalStorage & SessionStorage**: Practical for persisting user data across sessions and forms.
- **Code reviews are powerful**: They highlight subtle issues even in working apps, reinforcing professional best practices.
- **Modular repo structure**: Keeping each challenge in its own folder makes the repo easy to navigate and present.
